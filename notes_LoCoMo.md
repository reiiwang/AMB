# 論文筆記：LoCoMo

**標題：** Evaluating Very Long-Term Conversational Memory of LLM Agents
**作者：** Adyasha Maharana, Dong-Ho Lee, Sergey Tulyakov, Mohit Bansal, Francesco Barbieri, Yuwei Fang
**發表：** 2024 | arXiv:2402.17753
**論文連結：** https://arxiv.org/pdf/2402.17753

---

## 問題與動機

現有對話記憶研究只在**極短範圍**內評估（~1,000 tokens、~5 個 session），無法反映真實的長期對話場景。現實中的對話可能跨越數月、數百輪，包含複雜的時間順序與因果關係。在長文本 LLM 與 RAG 快速發展的背景下，卻沒有系統性的評估指出這些系統是否真的能維持長期對話記憶。

---

## 核心貢獻

1. **LoCoMo 資料集**：50 組超長期對話，平均 300 輪、9,209 tokens、19.3 個 session，模擬跨越數月的對話歷史，包含多模態（圖片分享）元素
2. **機器-人工生成 Pipeline**：以 persona、時間事件圖、顯式記憶系統驅動的 LLM agent 對話生成架構，並經人工驗證
3. **三項評估任務**：問答（5 種推理類型）、事件摘要、多模態對話生成

---

## 資料集情境

LoCoMo 模擬**兩位朋友之間的長期日常對話**。兩個虛擬角色各有完整人物背景（職業、個性、興趣），在數個月的時間內跨越多個 session 自然交流，內容涵蓋：

- 工作與職涯變動（換工作、升職、辭職）
- 個人生活事件（搬家、旅行、家庭事務）
- 日常閒聊（分享圖片、討論興趣愛好）
- 情感與人際關係變化

對話的時間感是核心設計：每段對話都有明確的模擬時間戳，事件之間存在因果與先後順序（例如「因為升職所以搬到新城市」），這使得時間推理成為必要能力。

> **一句話情境**：兩位擁有豐富人生經歷的朋友，透過通訊軟體進行跨越半年的日常聊天，偶爾分享圖片。

---

## 資料集建構

### Agent 架構
每個虛擬 agent 包含：
- **Persona**：從 MSC 資料集的角色描述擴展成豐富的人物設定
- **時間事件圖（Temporal Event Graph）**：每個 agent 最多 25 個因果相連的生活事件，模擬 6–12 個月的時間跨度（例如：「開始新工作」→「升職」→「搬家」）
- **短期記憶**：每個 session 結束後生成的摘要
- **長期記憶**：從對話輪次提取的原子級觀察事實（atomic observations），供後續 session 取回使用
- **多模態能力**：agent 可分享圖片並生成反應

### 生成流程
1. 每輪對話根據 persona、近期對話歷史、取回的長期觀察記錄、時間事件圖中的插入事件來生成
2. 每個 session 結束後生成摘要存入短期記憶
3. 對話輪次被分解為原子觀察記錄，供未來 session 取回

### 人工驗證
- 修正長程不一致性
- ~19% 的生成圖片需要替換或移除
- 約 15% 的對話輪次經人工編輯

### 資料集規模

| 指標 | 數值 |
|------|------|
| 對話組數 | 50 |
| 平均輪次/對話 | 300 |
| 平均 tokens/對話 | 9,209 |
| 平均 session 數/對話 | 19.3 |
| 時間跨度 | 數個月 |

---

## 三項評估任務與指標

### Task 1：問答（QA）
共 **7,512 題**，分為五種推理類型：

| 類型 | 比例 | 說明 |
|------|------|------|
| Single-hop | 36.0% | 單一 session 內可回答 |
| Multi-hop | 14.6% | 需整合多個 session 的資訊 |
| Temporal | 20.6% | 時間推理（先後順序、時間長度） |
| Open-domain | 3.9% | 需結合對話外的世界知識 |
| Adversarial | 24.9% | **無法回答**的問題，測試模型是否會幻覺而非正確拒答 |

**指標：** F1 score（帶正規化的部分匹配）

### Task 2：事件摘要
- 在指定時間範圍內摘要發生的事件
- **指標：** FactScore（原子事實的 Precision/Recall）+ ROUGE

### Task 3：多模態對話生成
- 生成包含圖片的對話回應，評估與 persona 的一致性
- **指標：** MMRelevance + BLEU 等 NLG 指標

---

## 主要實驗結果

### QA 結果

| 方法 | Overall F1 |
|------|-----------|
| Human | 87.9% |
| GPT-3.5-turbo-16K（長文本） | 37.8% |
| RAG（觀察記錄取回） | **41.4%** |
| RAG（session 摘要取回） | 32.5% |
| RAG（原始對話取回） | 31.7% |
| GPT-4-turbo（4K context） | 32.1% |
| 其他基礎模型平均（4K） | 22.4% |

**關鍵發現：**
- **長文本模型反而有害**：adversarial 題目 F1 從 22–70% 暴跌至 2.1%，顯示更多 context 大幅增加幻覺
- **時間推理仍是死穴**：長文本模型僅達 20.3–25.0%，人類達 92.6%，差距高達 73%
- **RAG + 原子觀察記錄最有效**（41.4%），取回超過 Top-5 後效果反而下降（噪音增加）

### 事件摘要結果
- GPT-3.5-turbo FactScore F1 最佳（45.9%）
- 長文本版本（16K）反而更差：Precision 下降 3.0%，Recall 下降 8.7%
- 五類錯誤：缺失事件連結、幻覺、誤解對話線索、說話者歸因錯誤、遺漏顯著事件

### 多模態對話生成
- RAG + 觀察記錄顯著優於無取回的基線
- 歷史長度增加時效能持續下降，RAG 可部分緩解

---

## 結論與啟示

1. **LLM 與人類差距巨大**：所有任務均顯著落後，時間推理差距最大（~73%）
2. **更多 context ≠ 更好效果**：長文本模型在摘要和 adversarial QA 上反而退步
3. **RAG + 結構化原子觀察記錄**是目前最有效的策略（F1 41.4%），優於直接給摘要或原始對話
4. **資訊密度比 context 視窗大小更重要**：取回內容的訊噪比決定效能
5. **時間因果推理**在長對話下仍是未解的核心挑戰

### 局限
- 依賴 LLM 合成對話（雖有人工驗證）
- 圖片視覺一致性受限
- 僅支援英文
- 使用閉源生成模型

---

## 與其他 Benchmark 的關係

LoCoMo（9K tokens, ~10K sessions）被後續論文引用時多數指出其 context 深度已不足以挑戰現代模型：
- MemoryAgentBench 明確指出 LoCoMo 的 context 過短（~10K tokens），並在其基礎上延伸到 103K–1.44M tokens
- MemBench 也將 LoCoMo 列為「只評估事實記憶、缺少反思記憶」的先前工作之一
